# ProxyParser 产品重构草案

正式设计规格已经沉淀到：

```text
docs/superpowers/specs/2026-05-14-proxyparser-product-redesign-design.md
```

当前确认方向：

- 技术栈保留：Bun + Elysia + SQLite + React + Vite。
- 产品重新设计，不沿用旧版本的信息架构和工作流。
- V1 定位为半公开模板生态：个人生成订阅是核心，模板和规则方案可以脱敏共享。
- 扩展订阅可以共享给指定用户或所有人。
- 目标是先做出完整可用的一版产品，而不是只修补当前 UI。

后续 agent 请以正式规格为准。

## 2026-05-14 实现进度

已完成并验证：

- 后端迁移与类型：模板可共享性、脱敏来源、共享授权、短期 Key 元数据。
- 扩展订阅渲染：最新成功快照回放、自动国家/地区分组、Proxies 总组、规则源挂载、RULE-SET 插入。
- 草稿导入：空草稿可直接粘贴外部订阅 URL，后台创建并同步外部订阅后绑定当前草稿。
- 共享与 Key：短期 Key 自定义有效期、列表、立即失效；长期 Key 重置；共享给所有用户或指定用户。
- 脱敏模板：从草稿提炼模板时可剥离真实节点，只保留规则、分组和后续操作。
- 前端主流程：扩展订阅向导、自动分组配置、规则源落点、脱敏模板按钮、订阅共享与 Key 控制台。

验证记录：

- `backend`: `bun test` 通过 14 个集成测试。
- `frontend`: `bun run build` 通过；Vite 仍提示默认 500 kB chunk warning。
- 浏览器走通注册、粘贴 mock 订阅 URL、预览、脱敏模板提炼、发布扩展订阅、共享给所有用户。
